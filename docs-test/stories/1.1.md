## Dev Notes
- Use `pyproject.toml` instead of `setup.py` as it's the modern Python packaging standard (PEP 518, PEP 621)
- Follow Python Packaging Authority (PyPA) best practices for project structure
- Package name should be lowercase with hyphens (e.g., `bubble-sort-tool`)
- Include development dependencies in `[project.optional-dependencies]` section
- Configure code quality tools (ruff, black, mypy) in `pyproject.toml`
- Ensure package can be installed in editable/development mode with `pip install -e .`
- Include type hints and mypy configuration for type checking
- Use `src/` layout instead of flat layout for better separation of concerns
- Follow PEP 8 style guidelines throughout the project
