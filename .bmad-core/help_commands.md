# James - Developer Commands

## Available Commands:

1. **help** - Show numbered list of available commands
2. **develop-story** - Execute story implementation with TDD workflow
   - Order: Read task → Implement → Write tests → Execute validations → Update checkboxes → Repeat
   - Updates only: Tasks/Subtasks checkboxes, Dev Agent Record sections, File List, Change Log
3. **explain** - Detailed explanation of implementation decisions for learning
4. **review-qa** - Run QA fixes task
5. **run-tests** - Execute linting and tests
6. **exit** - Exit developer persona

## Usage:
Type * followed by command name (e.g., *develop-story)
