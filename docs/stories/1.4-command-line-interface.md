# Story 1.4: Command-Line Interface

## Status
**Status:** Ready for Development

## Story
**As a** user,
**I want** a command-line interface to test the bubble sort algorithm,
**so that** I can easily demonstrate and experiment with the implementation.

## Acceptance Criteria
1. Create CLI entry point that accepts arrays as command-line arguments
2. Support reading arrays from files or standard input
3. Provide options for different output formats (sorted array, detailed process)
4. Include help documentation and usage examples
5. Handle command-line errors gracefully with helpful error messages
6. Support both interactive and batch modes of operation

## Dev Notes
This story creates a user-friendly command-line interface for the bubble sort algorithm. The CLI should be intuitive and provide multiple ways to interact with the algorithm for both educational and practical purposes.

CLI design considerations:
- Support multiple input methods: args, files, stdin
- Provide clear output options
- Educational value: show the process, not just results
- Error messages should be helpful to users

### Testing Standards
- Test CLI using pytest with subprocess testing
- Test all input methods (args, file, stdin)
- Test error handling scenarios
- Test help documentation is accessible
- Validate output formats match specifications

### Entry Point Configuration
- Entry point name suggestion: bubble-sort
- Configure in pyproject.toml [project.scripts] section
- Main module: src.cli or src.main

### Usage Examples
```bash
# Basic usage
bubble-sort 5 3 8 1 9

# From file
bubble-sort --file input.txt

# Detailed output
bubble-sort --format detailed 5 3 8 1 9

# Interactive mode
bubble-sort --interactive
```
