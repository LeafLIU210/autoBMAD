## Dev Notes
**Source:** Epic 1 - Core Algorithm Foundation

This story establishes the foundational project structure for the bubble sort implementation project. The project should follow Python best practices with proper packaging, modern tooling, and CI/CD automation.

**Project Context:**
- Focus on educational implementation of bubble sort algorithm
- Target audience: Students, educators, and developers learning sorting algorithms
- Priority on code clarity and documentation over optimization
- Must follow PEP 8 style guidelines
- All code must include type hints

**Testing Standards:**
- Use pytest as the testing framework
- Test file location: tests/ directory
- Aim for >95% code coverage
- Include parameterized tests for multiple scenarios

---

## Testing
**Test File Location:** tests/

**Testing Standards:**
- Use pytest framework
- Include unit tests for all functions
- Use parameterized tests for multiple input scenarios
- Test edge cases (empty lists, single elements, etc.)
- Achieve >95% code coverage
- Include both positive and negative test cases
